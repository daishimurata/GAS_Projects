# Gemini向け：LINE WORKS統合システム利用ガイド

**システム名**: LINE WORKS × Google Workspace 統合システム  
**バージョン**: 1.1.0  
**最終更新**: 2025年11月30日

---

## 🤖 このシステムについて

このシステムは、LINE WORKSのカレンダーとチャット履歴を自動的にGoogle Workspace（Googleカレンダー・Googleドライブ）に同期し、あなた（Gemini）が検索・分析できるようにしています。

---

## 📊 アクセス可能なデータ

### 1. Googleカレンダー（予定情報）

**カレンダーID**: `d.murata@izaya.llc`

**データ内容**:
- LINE WORKSの全メンバーのカレンダー予定が統合されています
- 予定のタイトルには `[メンバー名]` が付いています
- 説明欄にはカレンダー所有者の情報が含まれています

**データ範囲**:
- 過去7日間〜未来60日間
- 1日4回更新（5:00、10:00、16:00、21:00）

**活用例**:
- 「明日の予定は？」
- 「今週の田中さんの予定を教えて」
- 「12月に会議は何件ありますか？」
- 「誰が明日休みですか？」

### 2. マスタースプレッドシート（チャット履歴）

**場所**: `/LINE WORKS統合ログ/チャットログ/マスターログ.xlsx`

**シート構成**:

#### シート「メッセージ一覧」
全チャットルームのメッセージを時系列で統合

| 列 | 内容 | 説明 |
|---|---|---|
| A | 日時 | メッセージ送信日時 |
| B | 送信者 | 送信者の名前 |
| C | ルーム名 | チャットルーム名 |
| D | メッセージ | メッセージ本文 |
| E | 添付ファイル | 添付ファイル数 |
| F | メッセージID | LINE WORKSのメッセージID |
| G | チャンネルID | LINE WORKSのチャンネルID |
| H | キーワード | 自動抽出されたキーワード |
| I | カテゴリ | 自動分類（会議/休暇/報告/質問/決定/問題/依頼/その他） |
| J | URL | メッセージURL |

#### シート「ルーム一覧」
チャットルームの一覧と同期状況

#### シート「日次サマリー」
各ルームの日次要約（主要トピック、重要決定事項など）

#### シート「検索インデックス」
キーワードベースの検索インデックス

**活用例**:
- 「昨日の営業部のチャットをまとめて」
- 「田中さんが今週報告した内容は？」
- 「プロジェクトAに関する決定事項を教えて」
- 「先月の会議の内容をまとめて」
- 「"休暇"というキーワードが含まれるメッセージを探して」

### 3. テキストログ（詳細な会話履歴）

**場所**: `/LINE WORKS統合ログ/チャットログ/`

**フォルダ構造**:

```
チャットログ/
├── 日次ログ/
│   ├── 2025-01/
│   │   ├── 2025-01-15_全体ログ.txt
│   │   └── ...
│   ├── 2025-11/
│   │   ├── 2025-11-28_全体ログ.txt
│   │   ├── 2025-11-29_全体ログ.txt
│   │   └── 2025-11-30_全体ログ.txt
│   └── ...
│
├── ルーム別ログ/
│   ├── 2025-01/
│   │   ├── 営業部チーム_履歴.txt
│   │   ├── 開発チーム_履歴.txt
│   │   └── ...
│   └── 2025-11/
│       └── ...
│
└── 添付ファイル/
    ├── 2025-01/
    └── 2025-11/
```

**特徴**:
- 月ごとにフォルダで整理されています（YYYY-MM形式）
- 日次ログ: 全ルームの1日のメッセージを統合
- ルーム別ログ: 特定ルームの詳細な会話履歴

**活用例**:
- 「2025年11月の営業部チームのログを確認して」
- 「先週の日次ログから重要な決定事項を抽出して」
- 「開発チームの最新の会話履歴を読んで」

## 🔍 データの読み方のヒント

### カレンダーデータの特徴

1. **タイトルの読み方**
   - 形式: `[カレンダー所有者名] イベント名`
   - 例: `[田中太郎] 営業会議`
   - これにより、誰の予定かすぐに判別できます

2. **説明欄の情報**
   - 元のイベント説明
   - `---` 以降にLINE WORKSからの同期情報
   - カレンダー所有者のメールアドレス

3. **時刻はすべて日本時間（JST）**

### チャットデータの特徴

1. **キーワード列（H列）の活用**
   - 重要なキーワードが自動抽出されています
   - 検索の手がかりとして活用できます
   - 例: 「会議, 決定, 承認」

2. **カテゴリ列（I列）の活用**
   - メッセージが自動分類されています
   - カテゴリ:
     - **会議**: 会議、ミーティング関連
     - **休暇連絡**: 休み、休暇の連絡
     - **報告**: 業務報告、完了報告
     - **質問**: 質問、問い合わせ
     - **決定事項**: 重要な決定
     - **問題**: 問題、エラー報告
     - **依頼**: お願い、依頼事項
     - **その他**: 上記以外

3. **日次サマリーシートの活用**
   - 各日の要約が記載されています
   - 長期間のデータを素早く把握するのに便利です

4. **最新データが上に来ます**
   - スプレッドシートは最新のメッセージが上（2行目以降）
   - 時系列で遡りたい場合は下に向かって読みます

### 3. キーワードとカテゴリを活用

**良い質問例**:

- 「"問題"カテゴリのメッセージを日付順に並べて」

### 4. 日次サマリーから全体像を把握

**良い質問例**:
- 「今月の日次サマリーから主要トピックを抽出して」
- 「先週の重要決定事項を日次サマリーからまとめて」

**良い質問例**:
- 「営業部チームのルーム別ログから、顧客訪問の報告を抽出して」
- 「田中さんが送信したメッセージで"報告"カテゴリのものは？」

## 📋 よくある質問パターンと回答方法

### Q: 「明日の予定は？」

**アクセスするデータ**: Googleカレンダー

**回答の構成**:
1. 明日の日付を確認
2. カレンダーから明日の予定を検索
3. 時刻順に整理
4. メンバーごとに分類して提示

### Q: 「昨日のチャットをまとめて」

**アクセスするデータ**: 
- マスタースプレッドシート（メッセージ一覧）
- または日次ログ（昨日の日付のファイル）

**回答の構成**:
1. 昨日の日付を確認
2. 該当日のメッセージを抽出
3. ルームごとに分類
4. 重要な決定事項やトピックを要約

### Q: 「田中さんの今週の活動は？」

**アクセスするデータ**:
- Googleカレンダー（予定）
- マスタースプレッドシート（チャット）

**回答の構成**:
1. 今週の日付範囲を確認
2. カレンダーから田中さんの予定を抽出
3. チャットログから田中さんのメッセージを抽出
4. 時系列または活動種類別に整理

### Q: 「プロジェクトAについて何が決まった？」

**アクセスするデータ**:
- マスタースプレッドシート（特にカテゴリ"決定事項"）
- キーワード検索

**回答の構成**:
1. "プロジェクトA"をキーワード検索
2. カテゴリ"決定事項"でフィルタ
3. 日付順に整理
4. 決定内容を要約

## 🔄 データの更新頻度

### 同期スケジュール

- **更新時刻**: 毎日 5:00、10:00、16:00、21:00（日本時間）
- **更新頻度**: 1日4回
- **最大遅延**: 最大6時間の遅延がある可能性

### データの鮮度を確認する方法

1. **カレンダー**: Googleカレンダーの最終更新時刻を確認
2. **スプレッドシート**: 「メッセージ一覧」シートの最上部（最新メッセージ）の日時
3. **システムログ**: `/システムログ/YYYY-MM/同期ログ.txt` の最終行

**注意**: 
リアルタイムではないため、直近6時間以内の情報は反映されていない可能性があります。

## ⚠️ データ解釈時の注意点

### 1. 重複イベントについて

- 複数のメンバーが同じ会議に参加している場合、それぞれのカレンダーからイベントが同期されます
- タイトルの `[メンバー名]` で識別してください

### 2. 削除されたデータ

- LINE WORKSで削除されたイベント・メッセージは次回同期時に削除されます
- 過去のログファイルには残っている場合があります

### 3. プライバシーへの配慮

- このデータには個人の予定や会話が含まれます
- 必要な情報のみを抽出し、プライバシーに配慮してください

### 4. データの精度

- キーワード抽出とカテゴリ分類は自動処理です
- 100%正確ではない可能性があります
- 重要な判断の際は、元のメッセージ本文を確認してください

## 📝 システム制約事項

### データ範囲の制約

- **カレンダー**: 過去7日〜未来60日のみ
- **チャット**: 初回同期時は過去30日、以降は増分同期
- **古いデータ**: 12ヶ月より古いログはアーカイブされる可能性

### パフォーマンスの制約

- 大量データの検索には時間がかかる場合があります
- 期間を絞った検索を推奨します

### データ形式の制約

- テキストログは読みやすさ重視のため、完全な構造化はされていません
- 正確な分析にはスプレッドシートの利用を推奨します

## 🆘 トラブルシューティング

### データが見つからない場合

1. **データの更新タイミングを確認**
   - 最終同期時刻を確認（最大6時間の遅延）

2. **検索範囲を広げる**
   - 日付範囲を広げて検索
   - キーワードを変更して検索

3. **複数のデータソースを確認**
   - スプレッドシートで見つからない場合、テキストログを確認
   - その逆も試す

### データの不整合がある場合

- 次回の同期（最大6時間後）で修正される可能性があります
- システムログで同期エラーがないか確認できます
